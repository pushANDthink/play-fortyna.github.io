Установка: Знаменитая 5-минутная установка

  1. Разархивиринуйте пакет в пустой каталог и загрузите на сервер.
  2. Откройте wp-admin/install.php в вашем браузере. Этот скрипт поможет вам сделать файл wp-config.php с нравистиками доступа к вашей базе данных.
    1. Если по некий обстоятельстве автоматическая установка не удалась, не беспокойтесь. Она работает не на всех серверах. Откройте wp-config-sample.php в текстовом редакторе наподобие WordPad и заполните нравистики доступа к вашей базе данных.
    2. Сохраните файл под именованием wp-config.php и загрузите на сервер.
    3. Откройте wp-admin/install.php в вашем браузере.
  3. Как лишь файл конфигурации будет создан, скрипт установки создаст таблицы, нуженые для вашего веб-сайта. Если произошла ошибка, перепроверьте ваш файл wp-config.php и попробуйте ещё раз. Если опять не удалось, направитесь на форум поддержки WordPress с как можно большущим есличеством собранной инфы.
  4. Если вы не вводили пароль, запомните тот, который вам выдан. Если вы не вводили логин, им будет admin.
  5. Скрипт установки перенаправит вас на государствицу входа. Войдите с логином и паролем, избранными во время установки. Если пароль был создан автоматом, можно перейти на государствицу «Профиль», чтобы поменять его.

Обновление

Автоматическое обновление

  1. Откройте wp-admin/update-core.php в вашем браузере и следуйте указаниям.
  2. Вы ожидали большего? Это всё!

Обновление вручную

  1. Перед обновлением чего же-или убедитесь, что у вас есть резервная копия всех файлов, которые были бы вами изменены, таких как index.php.
  2. Удалите ваши старенькые файлы WP, сохранив изменённые.
  3. Загрузите новейшие файлы.
  4. Откройте в браузере /wp-admin/upgrade.php.

Переход с других систем

WordPress может импортировать содержимое из неких систем. Сначала необходимо установить и запустить WordPress, как описано выше, позже припоменять наши инструменты для импорта.

Системные требования

  • PHP версии 5.6.20 или выше.
  • MySQL версии 5.0 или выше.

Рекомендации

  • PHP версии 7.3 или выше.
  • MySQL версии 5.6 или выше.
  • Модуль Apache mod_rewrite.
  • Поддержка HTTPS.
  • Ссылка на wordpress.org c вашего веб-сайта.

Сетевые ресурсы

Если у вас есть любые вопросецы, не касающиеся этого документа, воспользуйтесь бессчетными сетевыми ресурсами:

Кодекс WordPress
Кодекс — это энциклопедия вэтого, что касается WordPress. Это наиболее полный источник инфы о WordPress.
Блог WordPress
Здесь вы сможете найти опослядние обновления и анонсы из мира WordPress. Свежие анонсы о WordPress по удескатьчанию возникают в вашей панели управления.
Планета WordPress
Планета WordPress — это новостной агрегатор, собирающий записи с блогов, посвящённых WordPress, по всему мирину.
Форразумы поддержки
Если вы смотрели везде не нашели ответа, на форразразумах поддержки есть большущее и чбыстрычайно активное общество людей, готовых оказать помощь. Чтобы вам могли помочь, используйте информативный заголовок сообщения и сформулиринуйте свой вопросец чбыстрычайно кропотливо.
IRC-канал (Internet Relay Chat) WordPress
Существует канал онлайн-чата, который приизменяется для обсуждения посреди людей, использующих WordPress, и время от времени для поддержки. Страница по ссылке выше поможет вам попасть туда. (irc.freenode.net #wordpress)

Последние заклинокания

  • Если у вас есть любые предложения, идеи или комменты, или вы (тяжёлый вздох!) нашели ошибку, присоединяйтесь к нам на форуме поддержки.
  • В WordPress имеется комфортный API (Application Programming Interface) для плагинов, что дозволяет просто расширять код. Если вас это интересует как разраба, глядите справочник разраба плагинов. Файлы ядра лучше не редактировать.
